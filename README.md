# BlastEX

Калькулятор параметров взрывных работ: подбор сетки скважин по модели Кузнецова и интерактивная схема заряда. В перспективе — перенос сметной логики из Excel (`Смета ТГ от 23.03.2026.xlsx`).

## Структура проекта

```
BlastEX/
├── Blast.py              # Технологический движок (Кузнецов, оптимизация q и ЛНС)
├── blast_hole.py         # Геометрия заряда одной скважины
├── blast_hole_viz.py     # Визуализация сечения скважины
├── app.py                # Веб-интерфейс (Streamlit)
├── max_bot.py            # Telegram-бот
├── requirements.txt
└── Смета ТГ от 23.03.2026.xlsx   # Источник экономической логики
```

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Модуль `Blast.py` — технология

Движок `BlastEngine` подбирает минимальный удельный расход `q` (кг/м³) при заданном пороге негабарита.

**Входные данные:**

| Класс | Поля |
|-------|------|
| `RockProperties` | порода, плотность, UCS, трещиноватость |
| `ExplosiveProperties` | название, плотность заряжания, теплота взрыва |
| `TargetParams` | размер негабарита, высота уступа, перебур, коэфф. сетки a/W |

**Выход оптимизации:** диаметр коронки, ЛНС `W`, сетка `a×b`, `x50`, `% негабарита`, удельный расход `q`.

Модель дробимости — **Кузнецов** + распределение **Розина–Рамблера** для оценки негабарита.

## Схема заряда скважины (этап 1 сметы)

После оптимизации в `app.py` появляется блок **«Схема заряда скважины»**: две колонки для сравнения вариантов (по умолчанию АСП / ЭВВ), как в Excel-смете.

### Формулы (`blast_hole.py`)

| Показатель | Формула |
|------------|---------|
| **Выход ВГМ** | `a × b × (глубина − перебур)` |
| **Длина заряда** | `глубина − недозаряд` |
| **Масса заряда** | `длина × π×d²/4 × ρ_ВВ` |
| **Удельный расход** | `масса / выход` |

На схеме: **недозаряд + заряд = полная глубина скважины**. Выход породы считается по эффективной высоте уступа `(глубина − перебур)`.

### Использование

1. Задайте параметры и нажмите **Рассчитать**.
2. Выберите вариант в **«Вариант для схемы заряда»**.
3. Меняйте **тип ВВ** и **недозаряд (верх скважины), м** — таблица и рисунок пересчитываются сразу.

### Сверка с Excel

При сетке 3.7×4.2, глубине 11 м, перебуре 1.5 м, коронке 152 мм:

| Показатель | Excel | BlastEX |
|------------|-------|---------|
| Выход АСП, м³ | 147.63 | 147.63 |
| Удельный АСП, кг/м³ | 0.866 | 0.876 |
| Заряд АСП, кг | 128 | 129 |
| Выход ЭВВ, м³ | 137.18 | 137.18 |
| Удельный ЭВВ, кг/м³ | 1.454 | 1.415 |
| Заряд ЭВВ, кг | 199 | 194 |

Небольшие расхождения по массе связаны с плотностью ВВ в справочнике (0.85 т/м³ vs 840 кг/м³ в смете).

## Логика Excel-сметы (`Смета ТГ от 23.03.2026.xlsx`)

Файл — **смета затрат на БВР**: калькулятор себестоимости и цены за м³ взорванной породы.

### Листы

| Группа | Листы |
|--------|-------|
| Расчётные сценарии | `125мм 0,68 0,78`, `смета расч. …`, `расчет 165 4,0х4,5` |
| Главный шаблон | `Расчет стоимости БВР` |
| Справочники | `ВВ`, `СВ`, `СИ`, `ТМЦ`, `ФОТ`, `5.амортизация новая`, `Объекты работ` |
| Тарификация | `Тарифная сетка`, `Расчет стоимости бурения` |

**Источник истины для переноса в код:** лист `Расчет стоимости БВР`.

### Блок 1 — технические исходные (левая часть сметы)

- Сетка `a × b`, глубина, перебур, диаметр заряда
- Выход, заряд, удельный расход, число скважин, погонаж бурения, масса ВВ

### Блок 2 — смета затрат (правая часть)

Расходы по **5 сценариям**: БВР сухие, БВР обводнённые, ВР сухие, ВР обводнённые, бурение.

**Переменные:**

- 1.1 — ВМ и СИ (ПВВ, ЭВВ, детонаторы, шнуры)
- 1.2 — буровой инструмент (коронка, ППУ, РК, обсадка)

**Постоянные:**

- 2.1 — хранение и доставка ВМ
- 2.2 — суточные, вахтовые, проживание
- 2.3 — ФОТ (+ отчисления 0.42% + 30% + резерв отпусков)
- 2.4 — ГСМ (бурение, мобилизация, пробег техники)
- 2.5 — амортизация
- 2.6 — общепроизводственные (ТО, ОСАГО, запчасти)

### Блок 3 — итоговая цена

```
Себестоимость = переменные + постоянные
+ АХР (5–10%)
+ рентабельность (5–10%)
= цена без НДС (руб/м³)
× 1.2 = цена с НДС
```

### Блок 4 — тарифная сетка

Регрессии от площади ячейки `S = a × b`:

```
Погонаж     = 70027 × S^(-1.006)
ВР на м³    = -3.4054 × S + 118.76
БВР         = 1913.2 × S^(-0.927)
Бурение/п.м.= (4.0554 × S + 800) × 1.2
Комплекс    = ВР + бурение на м³
```

### Блок 5 — стоимость бурения

Лист `Расчет стоимости бурения` → **цена за 1 п.м.** (ячейка D41): смены, амортизация БУ, расходники, ФОТ, ДТ, накладные, прибыль 20%. Подставляется в главную смету.

### Связь с BlastEX

| Модуль | Роль |
|--------|------|
| `Blast.py` | технология: `W`, `q`, `a×b`, диаметр, негабарит |
| Excel / будущий `cost/` | экономика: полная себестоимость и цена руб/м³ |

## План поэтапного переноса сметы в код

Каждый блок — отдельный файл + документ в `docs/cost/`.

```
cost/
├── models.py           # этап 0 — dataclass'ы
├── geometry.py         # этап 1 — скважины, заряд, удельный  ← частично: blast_hole.py
├── materials.py        # этап 2 — ВМ и СИ
├── drill_tools.py      # этап 3 — буровой инструмент
├── drilling.py         # этап 4 — стоимость бурения (руб/п.м.)
├── labor.py            # этап 5 — ФОТ
├── fuel.py             # этап 6 — ГСМ
├── depreciation.py     # этап 7 — амортизация
├── overhead.py         # этап 8 — хранение, суточные, общепроизв.
├── pricing.py          # этап 9 — сборка сметы, цена руб/м³
├── tariff_grid.py      # этап 10 — тарифная сетка (опционально)
└── engine.py           # этап 11 — CostEngine (фасад)
```

### Порядок реализации

| # | Модуль | Зависит от | Excel-лист |
|---|--------|------------|------------|
| 0 | `models` | — | — |

| 2 | `materials` | geometry | ВВ, СВ, СИ |
| 3 | `drill_tools` | geometry | ТМЦ |
| 4 | `drilling` | geometry | Расчет стоимости бурения |

| 10 | `tariff_grid` | geometry | Тарифная сетка |
| 11 | `engine` + UI | все | — |

### Шаблон работы на каждом этапе

1. `docs/cost/NN_*.md` — формулы, ссылки на ячейки Excel, входы/выходы
2. Модуль `.py` — расчёт без UI
3. Тест — сверка 1–3 кейсов с Excel
4. Интеграция в `app.py` / `max_bot.py`

### Решения, которые нужно зафиксировать перед кодом

1. **Справочники цен** — YAML/JSON в репозитории или hardcode на первом этапе
2. **Сценарии** — все 5 колонок сметы или только «комплекс БВР + бурение»
3. **Эталон Excel** — `Расчет стоимости БВР` (старые листы — для сверки, не один в один)
4. **Связка с Blast** — смета получает готовые `W`, `q`, `a`, `b` от `optimize_blast`
