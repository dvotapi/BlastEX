# BlastEX

Калькулятор параметров взрывных работ и сметы БВР: подбор сетки скважин по модели Кузнецова, интерактивная схема заряда, расчёт стоимости бурения и агрегированная смета по сценариям.

## Возможности

- **Технологический расчёт** — оптимизация удельного расхода ВВ, ЛНС, сетки скважин, оценка негабарита.
- **Смета** — переменные и постоянные затраты, ФОТ, стоимость бурения, несколько сценариев (`drill_blast`, `blasting`, `drilling` и др.).
- **Справочники команды** — породы, ВВ, амортизация ОС, объекты работ, бурстанки, номенклатура и цены; сохранение в `data/teams/{id}/`.
- **REST API** — расчёты БВР и сметы без Streamlit UI.
- **Админ-режим** — редактирование справочников по паролю.

## Быстрый старт

### 1. Виртуальное окружение

```bash
cd BlastEX
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Веб-интерфейс (Streamlit)

```bash
source .venv/bin/activate
streamlit run app.py
```

Откроется `http://localhost:8501`.

**Пароль администратора** (для редактирования справочников):

```bash
cp .streamlit/secrets.toml.example .streamlit/secrets.toml
# отредактируйте admin_password в .streamlit/secrets.toml
```

Или через переменную окружения:

```bash
export BLASTEX_ADMIN_PASSWORD="ваш-пароль"
streamlit run app.py
```

После смены пароля или создания `secrets.toml` перезапустите Streamlit.

### 3. REST API (опционально)

```bash
source .venv/bin/activate
pip install -r requirements-api.txt
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

- Swagger UI: `http://localhost:8000/docs`
- Health: `http://localhost:8000/health`

Через Docker (только API):

```bash
docker compose up --build
```

### Справочники

Подвкладки на **«Справочники»**:

| Раздел | Данные |
|--------|--------|
| Горные породы | Плотность, UCS, трещиноватость |
| Взрывчатые вещества | Плотность, теплота, подпись на схеме |
| Амортизация ОС | Стоимость, СПИ, смены; норма в смену считается автоматически |
| Объекты и станки | Мобилизация, ДТ, амортизация и расход топлива БУ |
| Номенклатура и цены | ВВ, СВ, СИ (цены без НДС) |
| Постоянные расходы | Разделы 2.1–2.6 сметы |

Справочники команды хранятся в `data/teams/{team_id}/references.json` и подставляются в расчёт БВР, бурения и сметы. Без входа администратора справочники доступны только для просмотра.

## Модуль `Blast.py` — технология

Движок `BlastEngine` подбирает минимальный удельный расход `q` (кг/м³) при заданном пороге негабарита.

**Входные данные:**

| Класс | Поля |
|-------|------|
| `RockProperties` | порода, плотность, UCS, трещиноватость |
| `ExplosiveProperties` | название, плотность заряжания, теплота взрыва |
| `TargetParams` | размер негабарита, высота уступа, перебур, коэфф. сетки a/W |

**Выход оптимизации:** диаметр коронки, ЛНС `W`, сетка `a×b`, `x50`, `% негабарита`, удельный расход `q`.

Модель дробимости — **Кузнецов** + распределение **Розина–Рамблера** для оценки негабарита.

Породы и ВВ в UI берутся из справочников команды (вкладка «Справочники»), а не из констант в коде.

## Схема заряда скважины

После оптимизации в `app.py` появляется блок **«Схема заряда скважины»**: две колонки для сравнения вариантов (по умолчанию АСП / ЭВВ), как в Excel-смете.

### Формулы (`blast_hole.py`)

| Показатель | Формула |
|------------|---------|
| **Выход ВГМ** | `a × b × (глубина − перебур)` |
| **Длина заряда** | `глубина − недозаряд` |
| **Масса заряда** | `длина × π×d²/4 × ρ_ВВ` |
| **Удельный расход** | `масса / выход` |

На схеме: **недозаряд + заряд = полная глубина скважины**. Выход породы считается по эффективной высоте уступа `(глубина − перебур)`.

### Использование

1. Задайте параметры и нажмите **Рассчитать**.
2. Выберите вариант в **«Вариант для схемы заряда»**.
3. Меняйте **тип ВВ** и **недозаряд (верх скважины), м** — таблица и рисунок пересчитываются сразу.

Реализованные модули:

| Модуль | Назначение |
|--------|------------|
| `geometry.py` | Скважины, заряд, удельный расход |

## Логика Excel-сметы (кратко)

**Блоки сметы:**

- **Переменные:** ВМ и СИ (1.1), буровой инструмент (1.2)
- **Постоянные:** хранение и доставка (2.1), суточные (2.2), ФОТ (2.3), ГСМ (2.4), амортизация (2.5), общепроизводственные (2.6)
- **Итог:** себестоимость + АХР + рентабельность → цена без НДС → × 1.2 с НДС

**Стоимость бурения** — отдельный лист «Расчет стоимости бурения» → цена за 1 п.м., подставляется в главную смету.
